### 2. 图文抓取与临时图床 (Image Hosting)
- 解析微信公众号文章 HTML，提取标题、正文和图片（注意微信图片的 `data-src` 属性）。
- **图片过滤**：剔除长宽过小（如 < 100px）的无用图片（如二维码、图标、分割线）。
- **图片中转**：由于豆包大模型需要公网可访问的图片 URL，而微信图片可能存在防盗链，你需要实现一个上传机制。
  - **首选方案**：将图片下载后，通过 API 上传至 **ttl.sh**（例如：`curl -T file.jpg https://ttl.sh/file.jpg?ttl=1h`）或 **Litterbox**，获取 1 小时有效期的临时直链。
  - 确保上传逻辑包含重试机制和超时处理。

### 3. 多模态大模型调用 (Doubao-vision-pro-32k)
- 必须按照文章的原始顺序，严格构建**“文本-图片-文本-图片”**交替的 JSON 数组（在 `content` 数组中交替使用 `input_text` 和 `input_image_url`）。
- 将获取到的临时图床直链喂给大模型。
- 提示词要求：让 AI 结合图片前后的文本上下文，深度解析图片的核心含义、数据图表的结论或关键信息。

### 4. Obsidian Markdown 与双链生成 (Bi-directional Links)
生成的 Markdown 必须符合高质量的 Obsidian 知识库标准：
- **Frontmatter**：包含 `title`、`date`、`source_url`、`tags`。
- **AI 视觉解析块**：将 AI 对图片的解析以引用块形式（`> 🤖 AI 视觉解析：...`）紧跟在对应图片下方。
- **智能双链 (核心)**：
  - 让 AI 在提取文章内容时，自动识别核心概念、行业术语、人名、公司名或重要理论，并将其转化为 Obsidian 双链格式（例如：`[[知识管理]]`、`[[大语言模型]]`）。
  - 在文章末尾生成一个 **“关联概念”** 区域，列出 3-5 个由本文延伸出的高价值双链标签。
